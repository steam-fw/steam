Look Mom! No Database!

Are you building a simple site which consists of mostly static HTML pages? There's nothing wrong with that. For many people, a database is unnecessary. Not to worry, Steam isn't going to force you to use one—it's entirely your decision.

The availability of web services has given us the freedom to rely upon external sites for dynamic content, and completely forego a database. Popular services include the Google Data APIs, Facebook Graph API, Twitter API, and the MediaWiki API.

That's great, but I'm trying to build a highly scalable, distributed system

Okay big shot, we've got you covered. Steam provides a simple interface to Zend Db which enables seamless transition to random selection load balancing using a single master-multiple slave configuration.

Wait, Steam isn't an OOP MVC Framework?

As much as we love using buzzwords to promote our software, it doesn't always make sense. Steam is beautifully organized, but it's not object oriented and it does not adhere to any existing architectural pattern. Steam offers architectural flexibility and relies heavily on static classes which are never instantiated. This has been determined to be a superior approach for web applications.
